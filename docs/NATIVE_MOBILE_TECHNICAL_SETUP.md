# Native Mobile Technical Setup

## Chosen stack

- Shared game core: JavaScript/TypeScript, SVG and Vite
- Native runtime: Capacitor v8
- iOS project: Xcode / Swift host generated by Capacitor
- Android project: Android Studio / Gradle host generated by Capacitor
- Compiler: Python
- Settings: Capacitor Preferences
- Progress file: Capacitor Filesystem
- Lifecycle: Capacitor App
- Haptics: Capacitor Haptics
- Share: Capacitor Share
- Launch UI: Capacitor Splash Screen and System/Status Bars
- Orientation: Capacitor Screen Orientation

## Initial dependency plan

```bash
npm install @capacitor/core @capacitor/cli
npm install @capacitor/ios @capacitor/android
npm install @capacitor/app @capacitor/preferences @capacitor/filesystem
npm install @capacitor/haptics @capacitor/share
npm install @capacitor/splash-screen @capacitor/status-bar
npm install @capacitor/screen-orientation
```

Then:

```bash
npx cap init
npx cap add ios
npx cap add android
npm run build
npx cap sync
```

Exact package versions must be locked together in `package-lock.json` and verified before implementation.

## Proposed Capacitor configuration

```ts
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.cogtech.toxicteddies',
  appName: 'Toxic Teddies: Arrow Escape',
  webDir: 'dist',
  bundledWebRuntime: false,
};

export default config;
```

Do not configure a remote `server.url` for production. Production content must come from the bundled Vite output.

## Native app bridge

Create `src/app/native-bridge.js` as the only application-facing wrapper around Capacitor plugins.

It should provide:

```text
isNative()
getPlatform()
onAppPause(callback)
onAppResume(callback)
hapticValid()
hapticBlocked()
hapticComplete()
loadSettings()
saveSettings(settings)
loadProgress()
saveProgress(progress)
shareResult(payload)
setSystemBars(theme)
lockPhonePortrait()
```

Game and UI modules should not import Capacitor plugins directly. This keeps native behavior testable and avoids platform calls spreading through the codebase.

## Save-file design

Preferences keys:

```text
settings-v1
onboarding-v1
last-level-v1
```

Filesystem file:

```text
toxic-teddies/save-state-v1.json
```

The save file includes:

```json
{
  "schema_version": 1,
  "app_version": "0.1.0",
  "completed": {},
  "active_session": {
    "level_id": "tt01_neutral",
    "level_version": 1,
    "compiler_version": "1.0.0",
    "removed_path_ids": []
  },
  "updated_at": "ISO-8601"
}
```

Writes should be atomic where practical: write a temporary file, then replace the current save after successful completion.

## Platform behavior

### iOS

- Respect safe-area insets.
- Test VoiceOver.
- Configure status-bar appearance.
- Use native share sheet.
- Use Taptic feedback through Capacitor Haptics.
- Save immediately when the app resigns active or enters background.
- Test TestFlight release configuration, not only simulator debug.

### Android

- Handle hardware/gesture Back correctly.
- Respect edge-to-edge system UI.
- Test TalkBack.
- Test gesture and three-button navigation.
- Build Android App Bundle for Play.
- Test pause/resume and process recreation.
- Test low-end device performance.

## Native build acceptance

The bootstrap is complete when:

- [ ] `ios/` exists and builds.
- [ ] `android/` exists and builds.
- [ ] The app uses local `dist` files.
- [ ] Toxic Toby opens on both platforms.
- [ ] Airplane-mode launch works.
- [ ] Haptics can be called without crashing.
- [ ] App pause/resume callbacks fire.
- [ ] Settings can be saved and loaded.
- [ ] A progress JSON file can be written and read.
- [ ] No signing secret is committed.
